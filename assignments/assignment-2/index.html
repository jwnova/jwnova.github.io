DOCUMENTATION

Like building a house, UX design has a working process from abstract to concrete, which can be broken down into five elements: strategy, scope, structure, skeleton, and surface (Munir). When we work with low-fidelity design is understood as a representation of the final design that shows only the elements relevant to the analysis, regardless of the support. When it comes to a sketch, the low-fidelity design is called a WireFrame. (Marron). Using wireframes can help programmers and designers communicate the architecture of the application they are building. Wireframes do not represent the final design, but they can save design time and reduce adjustment efforts.

For this project, we develop a series of Wireframes of the FilmFreeway website to describe the user's journey to submit a film in one of the Festivals that FilmFreeway offers. On the home page of the FilmFreeway website, the user can find the logo of the page on the left and the navigation menu with four hyperlinks on the right. If the user scrolls down, we can see more information about this website, but the main point of interest of the user is the Call to action red button "Browse Festivals.

After clicking on the CTA (Call to Action), the core page of the website is entered, which is also the information page. The users can set a range of filters on this page to search in detail for the film festival that meets their needs. After obtaining the search results, users can browse the detailed information about the film festival or submit a proposal. The core of the filter are options; we can see checkboxes used on this page. In addition, information cards for individual festivals appear as a group, which includes festival icons, thumbnail information, and buttons for users to interact with.

Once the user finds the film festival that meets their needs and clicks on the button "View Festival" to browse detailed information, the site redirects to the informative page of the selected film festival. On this page, the text content and images are the main elements of the page. If the user wants to continue submitting a proposal once it's clicked on the CAT green button "Submit Now", the user cannot continue until registered or logged in. The log-in page mainly consists of buttons and text boxes as the primary functional group. The text boxes contain default text to instruct the user and guide the user to perform subsequent operations. Users who do not already have an account can click the sign-up button to enter the registration page. We find similarities to the log-in page; different controls are set to meet user registration requirements to simplify user operations and increase user choices. Users can choose to authorize registration through their Facebook account or Google account or register a new account using their email.

After log-in, the user can click on the CAT "Submit Now" to submit their projects within the film festival. The web page will pop up a prompt box to remind users to add projects. Users need to upload their project information and related content on the page for adding projects. This whole page is a form in which the elements are mainly text boxes, combo boxes, checkboxes. Like the previous content, the text boxes contain default text to guide the user to fill in the relevant information. Once all the information is completed, the user can click on the CAT "Save project".

The following pop-up box contains two paragraphs of text with two hyperlinks that instruct the user to either go to the project page to upload the project or return to the search page to find the festival again. If the user clicks on the hyperlink "Add Project File", the user will be directed to a new page to upload files related to the project. The page is made up of tab bars and many buttons. Users can click different buttons to upload additional files; by clicking other tabs, the user can access the view tab and edit previously uploaded content or select different privacy settings. On the other hand, if the user clicks on the hyperlink." Browse Festivals", the user will be redirected to the information page to search for the festival again. Once the user finds the film festival and clicks on the button "View Festival", the informative page of the festival will appear; there, the user needs to click on the button "Submit Now". The festival's payment options page will pop up with checkboxes, where the user can view different prices and choose different services to suit their needs.

After selecting the base fee, the user is presented with a payment page to choose value-added services and fill in the payment method. Base fees, value-added services, payment methods belong to different groups on this page but are placed on the same page because they are in the same category of interactions, which helps users review their payments. The main elements of the page are buttons and text boxes.

The last steps that we designed were the final payment information page. The user needs to complete the detailed payment information in the text box according to the prompts provided and complete the payment and the confirmation payment as a pop up with the text box.
